# 05 · Google Business Profile — 12-Week Content Calendar

**Goal**: weekly GBP post for the duration of the 3-month trial. Google's local pack ranking algorithm rewards GBP activity; a profile that posts every week consistently outranks one with the same review count but no posts.

Each post is short (100–300 characters), uses one image (drawn from the existing project library where possible), and ends with a concrete CTA.

**Queue all 12 in advance** so the workflow runs without ad-hoc effort during the trial.

---

## Post types — round-robin to keep variety

1. **Project showcase** (every 3rd week) — pull from the live `/projects` pages
2. **Seasonal/timely tip** (every 4th week) — Brisbane storms, summer A/C, winter electrical, etc.
3. **Service highlight** (every 4th week) — link to one of the 7 service pages
4. **Compliance reminder** (every 4th week) — QLD smoke alarm 2027, RCD testing, etc.
5. **Team highlight** (every 4th week) — drives traffic to /team

## Scheduling

GBP supports scheduling via the dashboard (Posts → Schedule). Bulk-schedule all 12 in one sitting:

1. Pick a fixed posting day (suggest **Monday 9am AEST** — typical small-business uplift window).
2. Paste each week's draft, add the image, schedule.
3. Done — 12 weeks of activity in 60 minutes of one-time work.

If GBP scheduling proves unreliable (Google reportedly drops scheduled posts occasionally), use a tool like Publer or SocialPilot which support GBP and re-publish automatically on failure.

---

## After Week 12

The trial review at Month 3 measures impact. If GBP-driven traffic is showing in GA4, double the cadence (twice weekly) for the post-trial care plan. If it's not moving the needle, drop to fortnightly and lean harder on review velocity.
